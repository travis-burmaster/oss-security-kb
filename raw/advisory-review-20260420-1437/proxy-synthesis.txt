### Security Posture Notes

- All three known advisories have been resolved in current releases. The latest version (8.5.10) is not affected by any of them.
- The two 2021 ReDoS issues are closely related (both in source-map handling) but are tracked under distinct CVEs with slightly different affected ranges. CVE-2021-23382 broadened the affected set to all versions `<7.0.36` (not just `>=7.0.0`).
- CVE-2023-44270 is an integrity issue (not availability) — it could allow an attacker to smuggle CSS rules past comment-based filtering. Impact depends on whether the consumer trusts PostCSS output for security boundaries.
- Extremely high download volume (~207M/week) makes even moderate-severity issues high-impact in aggregate.
- No public security policy (`SECURITY.md`) was included in the evidence; its existence is **uncertain**.

### Recommendations

1. **Upgrade to ≥8.4.31** (preferably latest 8.5.x) to resolve all known advisories.
2. Projects still on the 7.x line should upgrade to at least **7.0.36** to address both ReDoS issues. Note that CVE-2023-44270 has no documented 7.x fix — the advisory marks `<8.4.31` as affected, implying **no patched 7.x version exists** for that issue.
3. If parsing untrusted CSS, treat PostCSS output as potentially attacker-influenced and apply independent validation downstream.
