# `ws` — npm Package Security KB Entry (Draft)

---

## 1. Package Summary

`ws` is a popular, lightweight WebSocket client and server implementation for Node.js, hosted at [github.com/websockets/ws](https://github.com/websockets/ws), with approximately 184 million weekly npm downloads (as of April 2026) and a latest published version of **8.20.0**.

## 3. Security Posture Notes

- **Responsible disclosure process exists.** SECURITY.md directs reporters to `security@3rd-Eden.com` and states that advisories are tracked via GitHub Security Advisories.
- **All five publicly known advisories have published fixes.** The latest version (8.20.0) post-dates the most recent fix (8.17.1 for CVE-2024-37890), so current users on the latest major branch should not be affected by any known advisory.
- **Historical vulnerability pattern is predominantly DoS.** Four of the five advisories are denial-of-service issues; one is an information-disclosure bug. No known remote-code-execution advisories exist in public records.
- **Input-parsing surfaces are the primary attack vector.** Advisories cluster around HTTP header parsing (`Sec-WebSocket-Extensions`, `Sec-WebSocket-Protocol`, generic header count) and message-size handling, suggesting these are the most security-sensitive code paths.
- **Mitigations documented by maintainers.** Release notes for CVE-2021-32640 recommend header-size limits; notes for CVE-2024-37890 recommend reducing maximum HTTP header size or adjusting `server.maxHeadersCount`.
- **Extremely high download count (~184 M/week)** amplifies the impact of any future vulnerability but also implies broad community scrutiny.
- **No evidence of supply-chain compromise, typo-squatting incidents, or maintainer-account takeover** was found in the reviewed public data.
