## 1. npm/cookie — Recommended Edits

**Rationale:** The existing page has two concrete deficiencies backed by this pass's evidence: stale download figures and unclear sourcing demarcation between audit-ingested advisory data and manually verified notes.

**Proposed edits:**

### a. Update download statistics
Replace the existing download figure with:

> Weekly downloads (as of this review pass): ~137,648,000. Current npm latest: 1.1.1.

### b. Clarify advisory sourcing
Add or revise a metadata line near the CVE-2024-47764 / GHSA-pxg6-pf52-xh8x entry:

> **Source:** This advisory is confirmed present in OSV's package-scoped query results for `npm/cookie` (GHSA-pxg6-pf52-xh8x / CVE-2024-47764). It is the **only** package-level advisory returned by OSV for this package as of this review pass. The vulnerability allows cookie name, path, or domain values to inject additional cookie attributes. Fixed in cookie 0.7.0. Current latest (1.1.1) is not affected.

**What NOT to add:** No new CVEs or advisories were found. Do not fabricate or speculate about additional vulnerabilities.

## 2. npm/express-session — Recommended Edits

**Rationale:** The existing stub lacks actionable dependency-chain context that is now confirmable from two public sources (OSV query results and upstream HISTORY.md). These are not direct advisories against express-session but are relevant transitive-risk notes.

**Proposed edits:**

Add a section titled **Dependency-Level Security Context** (or similar):

> **Direct advisories:** No package-scoped vulnerabilities are returned by OSV for `npm/express-session` as of this review pass.
>
> **Transitive dependency notes (from upstream HISTORY.md):**
>
> - **cookie ≥0.7.0 adoption (express-session 1.18.1):** express-session 1.18.1 updated its `cookie` dependency to 0.7.0, which includes the fix for GHSA-pxg6-pf52-xh8x / CVE-2024-47764 (cookie attribute injection via name/path/domain). Users on express-session <1.18.1 inherit exposure to this cookie-level vulnerability.
>
> - **cookie-signature 1.0.4 adoption (express-session 1.5.2):** express-session 1.5.2 updated its `cookie-signature` dependency to 1.0.4, which addressed a timing attack in signature comparison (GHSA-92vm-wfm5-mxvv / CVE-2016-1000236). Users on express-session <1.5.2 inherit exposure to this cookie-signature-level vulnerability.
>
> **Current status:** npm latest is 1.19.0 (weekly downloads: ~3,598,000), which includes both of the above dependency updates.

**Scoping note to include:**

> These are not advisories against express-session itself. They are documented here to support dependency-aware risk assessment. The advisory details and fix versions are sourced from the respective upstream packages' public records.
