## 2. `micromatch`

### Page: `packages/npm/micromatch.md`

```markdown
# micromatch

Glob matching library for JavaScript, optimised for speed.

| Field | Value |
|---|---|
| **Package** | [micromatch](https://www.npmjs.com/package/micromatch) |
| **Repository** | [micromatch/micromatch](https://github.com/micromatch/micromatch) |
| **Ecosystem** | npm |
| **Latest** | 4.0.8 |
| **Weekly Downloads** | ~142M |
| **Status** | ✅ No known active vulnerabilities on latest |

## Audit History

| Date | Event |
|---|---|
| 2024-04 | ReDoS vulnerability reported ([micromatch#243](https://github.com/micromatch/micromatch/issues/243)); CVE-2024-4067 assigned |
| 2024-05 | Initial fix attempt via [PR #247](https://github.com/micromatch/micromatch/pull/247) |
| 2024-08 | Further fix in [PR #266](https://github.com/micromatch/micromatch/pull/266) (commits [03aa805](https://github.com/micromatch/micromatch/commit/03aa8052171e878897eee5d7bb2ae0ae83ec2ade), [500d5d6](https://github.com/micromatch/micromatch/commit/500d5d6f42f0e8dfa1cb5464c6cb420b1b6aaaa0)) |
| 2024-08-21 | Advisory reviewed on GitHub Advisory Database |
| 2024-08 | v4.0.8 released with fix ([release](https://github.com/micromatch/micromatch/releases/tag/4.0.8)) |

## Vulnerabilities

| ID | Aliases | Severity | Affected | Fixed | Summary |
|---|---|---|---|---|---|
| [GHSA-952p-6rrq-rcjv](https://github.com/advisories/GHSA-952p-6rrq-rcjv) | CVE-2024-4067 | **MODERATE** | `<4.0.8` | 4.0.8 | ReDoS in `micromatch.braces()` via greedy `.*` pattern in `index.js` |

## Posture Notes

- The ReDoS occurs in `micromatch.braces()` ([index.js L448](https://github.com/micromatch/micromatch/blob/2c56a8604b68c1099e7bc0f807ce0865a339747a/index.js#L448)) where a greedy `.*` pattern causes catastrophic backtracking on crafted input.
- All versions prior to 4.0.8 are affected — this is a **wide blast radius** given ~142M weekly downloads and deep dependency trees (e.g., via `fast-glob`, `chokidar`, `jest`).
- The initial PR (#247) was merged but **did not fully resolve** the issue; the follow-up PR (#266) completed the fix in 4.0.8.
- Only exploitable when untrusted input is passed to braces expansion.
- Assessed as MODERATE severity (CVSS); practical exploitability depends on whether glob patterns originate from user input.
