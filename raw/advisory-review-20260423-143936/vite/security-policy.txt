Source: https://github.com/vitejs/vite/security/policy
Fetched: 2026-04-23 via public web fetch / GitHub security policy page

Key excerpts used for KB curation:
- "The dev server and preview server must treat all inbound network requests as potentially hostile. This includes malformed requests."
- "Configuration and plugins" plus imported config code and transitive dependencies are treated as developer-authored and therefore trusted.
- Project files, assets, and installed packages referenced by the project are treated as trusted.
- The page explicitly distinguishes in-scope Vite vulnerabilities from issues outside the threat model, while noting some out-of-scope issues may still be fixed.
- Example in-scope issue classes named on the page include crafted URLs that return file contents outside the `server.fs` boundary, bypassable `server.fs.deny`, missing or bypassable origin / host validation for dev-server endpoints, and XSS from code Vite injects into production bundles.
