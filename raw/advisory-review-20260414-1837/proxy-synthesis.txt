## Package Summary

`ws` is the dominant WebSocket implementation for Node.js, with approximately 184 million weekly npm downloads as of mid-April 2026. The latest observed version is 8.20.0. It is maintained under the `websockets` GitHub organization and has a published SECURITY.md with a dedicated security contact (security@3rd-Eden.com). The project has a documented history of five publicly disclosed vulnerabilities spanning 2016–2024, all of which have published fixes across affected release lines.

## Posture Notes

- **Massive adoption surface:** ~184M weekly downloads makes `ws` one of the most depended-upon packages in the npm ecosystem; any vulnerability has an outsized blast radius.
- **Known advisory set is modest and fully patched:** Five CVEs/GHSAs spanning 2016–2024 (memory disclosure, DoS via large messages, DoS via crafted headers, ReDoS, DoS via excessive HTTP headers). All have published fixes across maintained branches.
- **Multi-branch fix discipline:** Advisories (e.g., CVE-2021-32640, CVE-2024-37890) show fixes backported to multiple active release lines (5.x, 6.x, 7.x, 8.x), indicating a mature maintenance posture.
- **Formal security policy exists:** SECURITY.md is present in the repo with a named contact, and the project's release history section explicitly enumerates past security events — a transparency positive.
- **Vulnerability pattern is protocol-parsing DoS:** Four of the five advisories are denial-of-service via malformed or excessive protocol-level input (headers, extensions, message size). This is a consistent attack surface theme worth noting for consumers.
- **No unpatched or embargoed issues observed in this pass:** The OSV/GitHub advisory set shows no open or unfixed advisories against the current 8.x line as of the evidence window.

## Open Questions

1. **Lag between 8.17.1 (CVE-2024-37890 fix) and current 8.20.0:** What functional or hardening changes landed in 8.18.0–8.20.0? Are any security-relevant but below the advisory threshold?
2. **Downstream exposure measurement:** Given the download volume, how many direct dependents still pin vulnerable pre-fix versions (e.g., ws <7.5.10 or <8.17.1)? This would inform ecosystem-level risk guidance.
3. **`server.maxHeadersCount` reliance:** CVE-2024-37890 was tied to Node.js's `server.maxHeadersCount` default. Has upstream `ws` adopted any independent header-count ceiling, or does safe behavior still depend on the consumer configuring this correctly?

## Recommendation

`ws` merits a knowledge-base page this pass. The combination of extreme adoption breadth (~184M weekly downloads), a clean and fully citable advisory history (five CVEs, all patched, all with GHSA identifiers), a documented security policy, and a consistent DoS-via-protocol-parsing attack pattern makes it a high-value, low-ambiguity candidate. All claims are directly traceable to public OSV/GHSA records, npm metadata, and the upstream SECURITY.md — no inference or fabrication is required. The page would serve as a reliable reference for dependency-review tooling and consumer risk assessment.
