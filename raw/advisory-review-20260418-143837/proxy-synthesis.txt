## Audit History Summary

No public third-party security audit report for Flask has been identified in the gathered evidence. Security fixes to date have been driven by maintainer-identified issues and community-reported vulnerabilities disclosed through GitHub Security Advisories (GHSA).

## Known Vulnerabilities

| ID(s) | GHSA | Severity | Summary | Fixed In | Notes |
|---|---|---|---|---|---|
| CVE-2018-1000656 / PYSEC-2018-66 | GHSA-562c-5r94-xh97 | Moderate–High | Crafted JSON payload can cause unexpected memory consumption (DoS) | See advisory | **Overlap note:** This advisory and PYSEC-2018-66 refer to the same underlying issue. |
| CVE-2019-1010083 / PYSEC-2019-179 | GHSA-5wv5-4vpf-pj6m | Moderate–High | Crafted JSON request can cause high CPU/memory usage (DoS) | See advisory | **Overlap note:** Distinct CVE from CVE-2018-1000656, but both concern DoS via crafted JSON input. The root-cause boundary between them is not fully delineated in public evidence; they may share contributing code paths. Treat conservatively as **two separate advisories** until further clarification. |
| CVE-2023-30861 / PYSEC-2023-62 | GHSA-m2qf-hxjv-5gpq | Important | Session cookie could be cached and served to other users when session is accessed/modified; missing `Vary: Cookie` header | **2.3.2** | CHANGES.rst: *"Set `Vary: Cookie` header when the session is accessed, modified, or refreshed."* |
| CVE-2025-47278 | GHSA-4grg-w6v8-c28g | Moderate | When `SECRET_KEY_FALLBACKS` (key rotation) is enabled, signing key selection order was incorrect, potentially using a fallback key to sign new sessions | **3.1.1** | CHANGES.rst: *"Fix signing key selection order when key rotation is enabled via `SECRET_KEY_FALLBACKS`."* |
| CVE-2026-27205 | GHSA-68rp-wp8r-4726 | Moderate | `Vary: Cookie` header not set for session operations that only access keys (e.g., `in`, `len`) — incomplete fix related to CVE-2023-30861 scope | **3.1.3** | CHANGES.rst: *"The session is marked as accessed for operations that only access the keys but not the values, such as `in` and `len`."* |

### CVE-2018-1000656 / CVE-2019-1010083 Overlap Note

Both advisories involve denial-of-service via crafted JSON payloads. Public descriptions suggest they may address overlapping but not identical attack vectors. In the absence of explicit maintainer clarification differentiating the root causes, this KB conservatively lists them as **separate entries** and flags the potential overlap. Contributors with deeper code-level insight are encouraged to clarify.
