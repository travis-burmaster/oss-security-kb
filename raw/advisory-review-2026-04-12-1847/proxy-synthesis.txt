# Proposed OSS KB Updates

## basic-ftp

### Current State Assessment
The KB page likely references some but not all of the three advisories affecting the 5.2.x line. All three are published in OSV/GHSA with CVE assignments and confirmed upstream fixes.

### Proposed Update

**Add/update three advisory entries:**

| Advisory | CVE | Vulnerability | Introduced | Fixed | Severity |
|---|---|---|---|---|---|
| GHSA-5rq4-664w-9x2c | CVE-2026-27699 | Path traversal in `downloadToDir` | < 5.2.0 | 5.2.0 | Review CVSS |
| GHSA-chqc-8p9q-pq6q | CVE-2026-39983 | FTP command injection via CRLF in commands | 5.2.0 | 5.2.1 | Review CVSS |
| GHSA-6v7q-wjvx-w8wg | *(pending or assigned)* | Incomplete CRLF injection protection (bypass of 5.2.1 fix) | 5.2.0 | 5.2.2 | Review CVSS |

**Key notes for KB text:**
- GHSA-chqc-8p9q-pq6q was *introduced* in 5.2.0 (the same release that fixed CVE-2026-27699), so versions 5.2.0 are simultaneously patched for the path traversal but newly vulnerable to CRLF injection.
- GHSA-6v7q-wjvx-w8wg represents an incomplete fix — the 5.2.1 CRLF mitigation was bypassable; full remediation requires ≥ 5.2.2.
- Both CRLF advisories are confirmed in upstream GitHub release notes for v5.2.1 and v5.2.2 respectively.

**Recommendation line:** Users should upgrade to ≥ 5.2.2 to address all three known vulnerabilities.

## path-to-regexp

### Current State Assessment
The KB page likely covers the two 2024-era advisories (CVE-2024-52798, CVE-2024-45296) and the manual-audit issue (#433). Three additional 2026-era advisories now exist.

### Proposed Update

**Add/update five advisory entries (keeping issue #433 as a separate, non-advisory item):**

| Advisory | CVE | Vulnerability | Fixed In | Affected Stream(s) |
|---|---|---|---|---|
| GHSA-rhx6-c78j-4q9w | CVE-2024-52798 | ReDoS via crafted input | 0.1.12 | 0.1.x |
| GHSA-9wv6-86v2-598j | CVE-2024-45296 | Backtracking ReDoS | 8.0.0 | < 8.0.0 |
| GHSA-37ch-88jc-xwx2 | CVE-2026-4867 | ReDoS via multiple route parameters | 0.1.13 | 0.1.x |
| GHSA-27v5-c462-wpq7 | CVE-2026-4923 | ReDoS via multiple wildcards | 8.4.0 | < 8.4.0 |
| GHSA-j3q9-mxjg-w52f | CVE-2026-4926 | DoS via sequential optional groups | 8.4.0 | < 8.4.0 |

**Key notes for KB text:**
- CVE-2026-4923 and CVE-2026-4926 are explicitly called out in upstream v8.4.0 release notes, which also describe architectural changes: restriction of wildcard backtracking and rejection of large optional-route combinations.
- The 0.1.x stream has had two separate ReDoS fixes (0.1.12 and 0.1.13); users on this legacy stream should ensure ≥ 0.1.13.
- The 8.x stream has had two rounds of ReDoS/DoS fixes (8.0.0 and 8.4.0); users should ensure ≥ 8.4.0.
- Manual-audit issue #433 remains a separate tracked item and should not be conflated with the published advisory set.

**Recommendation line:**
- Legacy 0.1.x users: upgrade to ≥ 0.1.13.
- Modern 8.x users: upgrade to ≥ 8.4.0.
- All other major versions between these streams should be evaluated for applicability or migrated to a supported release.
